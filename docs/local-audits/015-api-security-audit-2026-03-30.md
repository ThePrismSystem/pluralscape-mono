## Findings

### High

1. **Password change session revocation** — `account.service.ts:259-269` revokes all sessions EXCEPT the current session on password change. A compromised old device retains access if password is changed on another device. Intentional UX decision but needs explicit documentation. **Follow-up bean created.**

### Medium

2. **Content-Type not enforced in parseJsonBody** — `apps/api/src/lib/parse-json-body.ts` does not check Content-Type header before parsing. Clients can send non-JSON Content-Type with JSON body and it will be accepted. **Fixed inline.**

3. **Missing Cache-Control on sensitive endpoints** — Login, register, sessions, and account endpoints return sensitive data (session tokens, recovery keys, account info) without `Cache-Control: no-store`. Risk: proxy/CDN caching of auth responses. **Fixed inline.**

4. **Unconstrained query param strings** — `packages/validation/src/note.ts:36` and `packages/validation/src/audit-log-query.ts:5` have optional string fields without `.max()` constraints. Low DoS impact but inconsistent with other schemas. **Fixed inline.**

### Low

5. **Idle timeout bypass for unknown session types** — `session-auth.ts:110` returns null idle timeout for session types that don't match known configs. Custom/unknown session types may bypass idle enforcement. **Follow-up bean created.**

6. **Force flag string comparison** — `apps/api/src/routes/fields/delete.ts:22` uses `=== "true"` string comparison for force query param instead of strict boolean parsing. **Fixed inline.**

7. **Dead ALLOWED_ORIGINS env var** — `env.ts:30` defines `ALLOWED_ORIGINS` but it's never used. Only `CORS_ORIGIN` is consulted. **Fixed inline.**

## Auth/Authz (api-sojx)

### Auth Middleware Coverage — PASS

All protected routes have auth middleware:
- `/v1/systems/*` — blanket `authMiddleware()` at `systems/index.ts:48`
- `/v1/account/*` — blanket `authMiddleware()` at `account/index.ts:21`
- `/v1/notifications/stream` — `authMiddleware()` at `stream.ts:32`
- `/v1/auth/sessions` — `authMiddleware()` at `sessions.ts:28`
- `/v1/auth/biometric` — `authMiddleware()` at `biometric.ts:17`

Public routes (register, login, password-reset, recovery-key) correctly unprotected.

### IDOR / Ownership Checks — PASS

All system-scoped services use `assertSystemOwnership(systemId, auth)` before any data access. Helper at `system-ownership.ts:14-17` checks `auth.ownedSystemIds.has(systemId)`, throws 404 (not 403) to prevent existence leakage.

### Session Management — PASS (with finding #1)

- Token entropy: 256 bits (32 bytes libsodium randomBytes)
- Storage: BLAKE2b hashed
- Expiration: Absolute TTL + idle timeout
- Revocation: Immediate via revoked flag

### Unauthenticated Route Data Leakage — PASS

- Login: generic "Invalid email or password" + timing equalization + dummy Argon2 on missing email
- Registration: returns fake session on duplicate email (indistinguishable from success)
- Password reset: generic "Invalid email or recovery key"

### Biometric / Device Transfer — PASS

- Biometric: requires auth + system setting, tokens session-scoped, replay prevented via `usedAt`
- Device transfer: cryptographic binding, rate-limited (3/hour), timeout enforced, no token reuse
